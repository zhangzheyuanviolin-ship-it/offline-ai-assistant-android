# Offline AI Assistant - 自测步骤

## 环境准备

### 前置条件
- Android 设备或模拟器（API 24+）
- Expo Go 应用（可从 Google Play 下载）
- 一个 GGUF 格式的 AI 模型文件

### 启动应用

```bash
cd /home/ubuntu/offline-ai-assistant
pnpm install
pnpm start
```

然后在 Android 设备上：
1. 打开 Expo Go 应用
2. 扫描显示的二维码
3. 等待应用加载

## 测试场景 1：模型导入和选择

### 步骤

1. **启动应用**
   - 应用启动后显示聊天屏幕
   - 顶部显示"No Model"（未选择模型）

2. **导入模型**
   - 点击聊天屏幕中间的"📥 Import Model"按钮
   - 选择一个 GGUF 模型文件（如 `model.gguf`）
   - 等待导入完成
   - 应该看到"Model imported successfully"提示

3. **验证模型列表**
   - 点击顶部的"📦 No Model"区域
   - 打开模型管理屏幕
   - 应该看到刚导入的模型在列表中
   - 显示模型文件大小和格式

4. **选择模型**
   - 在模型列表中点击"Select"按钮
   - 应该看到"Model selected"提示
   - 返回聊天屏幕后，顶部应显示模型名称

### 预期结果
- ✓ 模型成功导入
- ✓ 模型显示在列表中
- ✓ 可以选择和切换模型
- ✓ 当前模型显示在顶部

## 测试场景 2：工具开关管理

### 步骤

1. **打开工具设置**
   - 在聊天屏幕点击工具开关按钮（🔍 Search、📁 Files、🎬 Media）

2. **测试 WebSearch 工具**
   - 看到"🔍 Web Search"开关
   - 点击开关禁用 WebSearch
   - 开关应该变灰
   - 点击开关重新启用
   - 开关应该变蓝

3. **测试 Files 工具**
   - 看到"📁 Files"开关和操作列表
   - 列表应显示 9 个文件操作
   - 切换开关状态

4. **测试 Media 工具**
   - 看到"🎬 Media"开关和操作列表
   - 列表应显示 5 个多媒体操作
   - 切换开关状态

5. **测试搜索引擎选择**
   - 启用 WebSearch 工具
   - 应该看到"Search Engine"选项
   - 可以在"🌍 International"和"🏠 Domestic"之间切换

### 预期结果
- ✓ 所有工具开关都能正常切换
- ✓ 开关状态在视觉上改变
- ✓ 搜索引擎选择正常工作
- ✓ 工具操作列表显示正确

## 测试场景 3：聊天界面和消息输入

### 步骤

1. **返回聊天屏幕**
   - 点击返回或底部导航返回聊天屏幕

2. **输入消息**
   - 在底部消息输入框中输入文本（如"Hello"）
   - 点击"Send"按钮
   - 消息应该出现在聊天窗口中
   - 消息应该显示在右侧（用户消息）
   - 消息应该有时间戳

3. **多条消息**
   - 继续输入多条消息
   - 所有消息应该按时间顺序显示
   - 消息列表应该自动滚动到最新消息

4. **消息显示**
   - 用户消息应该显示在右侧，背景为蓝色
   - 消息应该有清晰的时间戳
   - 消息应该可读且格式正确

### 预期结果
- ✓ 消息输入和发送正常
- ✓ 消息显示在正确的位置
- ✓ 消息有时间戳
- ✓ 消息列表自动滚动

## 测试场景 4：日志系统

### 步骤

1. **打开日志屏幕**
   - 在聊天屏幕点击"📋 Logs"按钮
   - 应该打开日志查看屏幕

2. **查看日志**
   - 如果没有工具调用，应该看到"No Logs Yet"提示
   - 日志列表应该显示最近 50 条记录

3. **日志详情**
   - 每条日志应该显示：
     - 工具名称
     - 成功/失败状态（绿色/红色）
     - 工具类别
     - 时间戳
     - 执行时间
     - 参数
     - 结果

4. **清空日志**
   - 点击右上角的"Clear"按钮
   - 应该弹出确认对话框
   - 确认后日志应该被清空
   - 应该看到"No Logs Yet"提示

### 预期结果
- ✓ 日志屏幕正常打开
- ✓ 日志显示格式正确
- ✓ 清空功能正常工作

5. **测试工具设置屏幕**
   - 打开工具设置屏幕
   - 所有开关和按钮都应该可被朗读
   - 开关状态应该可以被朗读（enabled/disabled）

### 预期结果
- ✓ TalkBack 能朗读所有关键元素

## 常见问题排查

### 问题：应用无法启动
**解决方案**：
1. 检查 Expo Go 版本是否最新
2. 检查网络连接
3. 查看 logcat：`adb logcat | grep -i "offline|error"`

### 问题：模型导入失败
**解决方案**：
1. 确认文件是有效的 GGUF 格式
2. 检查文件权限
3. 检查存储空间是否充足

### 问题：消息无法发送
**解决方案**：
1. 确认已选择模型
2. 检查消息不为空
3. 重启应用

### 问题：TalkBack 无法朗读
**解决方案**：
1. 确认 TalkBack 已启用
2. 检查音量设置
3. 尝试重启 TalkBack
